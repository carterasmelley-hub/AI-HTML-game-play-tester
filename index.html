Hosting Static GitHub Pages

Step 1

Add local HTML files

Put a game in games/your-game.html or games/your-game/index.html. Local files are the best default for fullscreen and asset reliability.

Step 2

Optional external links

Already hosting a game on GitHub Pages? Add that URL to data/external-games.json and it will show up in the same arcade.

Step 3

Push to GitHub Pages

The included workflow regenerates the catalog and deploys the site when you push to main or master.

Browser Uploads

Drag a game package right onto the page.

This works for testing on the site itself. Drop a full game folder, or an index.html plus its asset files, and the site will save the package in your browser on this device.

Browser packages can now include nearby audio, image, CSS, and JS files. If something is still missing, the upload will warn you which referenced files were not included.
